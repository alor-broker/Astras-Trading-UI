# terminal-widgets-lib

`terminal-widgets-lib` содержит общие виджеты, доступные приложениям терминала. Виджет — элемент дашборда, который пользователь может добавить или удалить, или какой-либо диалог.

## Назначение

Используй этот проект для виджетов и частей виджетов, которые могут быть доступны нескольким приложениям.

Не размещай здесь:

- код конкретного приложения;
- импорты из приложений репозитория;
- общие сервисы, которые нужны нескольким виджетам или приложениям.

Если сервис используется несколькими виджетами или приложением, размещай его в `projects/terminal-core-lib`.

## Границы импортов

Допустимо:

- импортировать Angular и внешние зависимости проекта;
- импортировать файлы внутри `terminal-widgets-lib`;
- импортировать из `projects/terminal-core-lib`.

Недопустимо:

- импортировать `desktop-terminal`, `mobile-terminal`, `admin-terminal` или другие приложения репозитория.

## Контекст дашборда

Виджеты из `terminal-widgets-lib` могут запускаться в разных приложениях. Реализация сервиса доступа к информации о текущем дашборде зависит от приложения, поэтому виджеты не должны напрямую использовать конкретные реализации вроде `DesktopDashboardContextService`.

Если виджету нужна информация о текущем дашборде, выбранном портфеле или инструментах, используй интерфейс `DashboardContextService` через injection token `DASHBOARD_CONTEXT_SERVICE` из `@terminal-core-lib/features/dashboard/services/dashboard-context-service.types`.

## Структура

| Папка | Что хранить |
| --- | --- |
| `src/assets` | Общее содержимое, доступное приложениям по `/assets` через настройки `angular.json` |
| `src/common` | Общие части виджетов и вспомогательные утилиты |
| `src/widgets` | Папки конкретных виджетов |

## Общие части виджетов

Папка `common` содержит элементы, которые рекомендуется использовать при создании новых виджетов:

- `WidgetSkeleton` — общий скелет компонента виджета;
- `WidgetHeader` и `WidgetHeaderInstrumentSwitch` — заголовок виджета и переключение инструмента;
- `WidgetSettings` — оболочка компонента настроек виджета;
- `WidgetBase` — базовый класс компонента виджета;
- `WidgetSettingsBase` — базовый класс компонента настроек виджета.

Пример виджета для ориентира: `projects/terminal-widgets-lib/src/widgets/orders-basket`.

## Структура папки виджета

Каждому виджету соответствует отдельная папка. Она может содержать:

| Элемент                    | Назначение                                                                                                                    |
|----------------------------|-------------------------------------------------------------------------------------------------------------------------------|
| `widget-settings.types.ts` | Типы настроек виджета. Каждый тип должен расширять базовый интерфейс `WidgetSettings`                                         |
| `<widget-name>-widget`     | Оболочка виджета: отображение, заголовок, переключение на настройки, создание default settings                                |
| `widgets`                  | Подпапки представлений, если у виджета несколько вариаций. Пример: `projects/terminal-widgets-lib/src/widgets/order-commands` |
| `types`                    | Типы, используемые только этим виджетом                                                                                       |
| `services`                 | Сервисы, используемые только этим виджетом                                                                                    |
| `utils`                    | Вспомогательные классы только для этого виджета                                                                               |
| `components`               | Внутренние компоненты виджета и компонент настроек, если пользователь может редактировать настройки                           |
| `styles`                   | Общие стили. Обычно миксины, которые могут быть включены в файлы стилей различных компонентов                                 |

Оболочку виджета рекомендуется наследовать от `WidgetBase`.

## Настройки виджета

Тип настроек виджета должен расширять базовый интерфейс `WidgetSettings`, если виджет является частью дашборда.

Если виджет зависит от текущего выбранного инструмента или имеет привязку к нему, интерфейс его настроек должен расширять `InstrumentKey` либо явно включать поля `symbol` и `exchange`.

Если виджет зависит от текущего выбранного портфеля или имеет привязку к нему, интерфейс его настроек должен явно включать поля `exchange` и `portfolio`.

## Добавление нового виджета

1. Создай папку виджета в `projects/terminal-widgets-lib/src/widgets`.
2. Добавь `widget-settings.types.ts` если виджет является частью дашборда (диалоги не имеют настроек)
3. Создай папку оболочки вида `<widget-name>-widget`.
4. В оболочке определи отображение виджета, заголовок, переход к настройкам и создание default settings, если настройки отсутствуют (используй WidgetSettingsFactoryHelper).
5. Вынеси внутреннее содержимое в `components`.
6. Размещай локальные типы, сервисы и утилиты в `types`, `services`, `utils`.
7. Если сервис нужен нескольким виджетам или приложению, перенеси его в `terminal-core-lib`.
8. Добавь metadata виджета в `projects/terminal-widgets-lib/src/assets/widgets-meta-config.json`.
9. Добавь виджет в `WIDGET_COMPONENT_REGISTRY` соответствующего приложения, если он доступен в этом приложении. Пример: `projects/desktop-terminal/src/app/widget-registry.ts`.

Регистрация в app registry нужна, чтобы приложения не включали неиспользуемые виджеты в bundle.

## AI checklist перед изменениями

- Виджет не импортирует приложения репозитория.
- Общая логика вынесена в `terminal-core-lib`, если она нужна нескольким виджетам или приложениям.
- Использованы общие части из `common`, если они подходят задаче.
- Settings type расширяет базовый `WidgetSettings`, если настройки есть.
- Settings type расширяет `InstrumentKey` или включает `symbol` и `exchange`, если виджет привязан к текущему инструменту.
- Settings type включает `exchange` и `portfolio`, если виджет привязан к текущему портфелю.
- Wrapper наследуется от `WidgetBase`, если нет причины не делать этого.
- Metadata добавлена или обновлена в `widgets-meta-config.json`, если виджет должен быть доступен пользователям.
- `WIDGET_COMPONENT_REGISTRY` обновлен в приложении, где виджет доступен.
- Доступ к контексту текущего дашборда выполняется через `DASHBOARD_CONTEXT_SERVICE`, а не через app-specific реализацию сервиса.
