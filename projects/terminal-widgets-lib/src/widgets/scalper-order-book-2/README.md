# scalper-orderbook-2

Высокопроизводительная версия виджета «Скальперский стакан» (`scalper-order-book`).
Цель — возможность открывать 100+ экземпляров виджета (в худшем случае каждый со своим
инструментом) без лагов взаимодействия.

Функциональность повторяет `scalper-order-book`. Отличие — архитектура отрисовки.

## Архитектура

### Разделение потока данных и отрисовки

- **Поток данных** полностью переиспользуется из `scalper-order-book`:
  `DataContextBuilder`, `PriceRowsStore`, `ScalperOrderBookDataProvider`,
  торговые команды, `ScalperCommandProcessorService`, `ScalperHotKeyCommandService`,
  сервисы настроек (включая общие per-instrument настройки группы `scob-ils`)
  и форма настроек.
- **Отрисовка** реализована на pixi.js и инкапсулирована в изолированный фасад
  вне Angular (`render/scalper-order-book-2-renderer.ts`). Внутри рендера нет rxjs
  подписок: данные передаются через сеттеры (`setRows`, `setOrders`, `setTrades`, ...),
  фасад помечает изменившиеся слои dirty-флагами.

### Синхронизация с экраном

Кадры планируются через `requestAnimationFrame` (`render/shared-render-engine.ts`):
если данные приходят чаще частоты обновления экрана, отрисовывается только последний
снимок. Виджеты без изменений не отрисовываются вообще.

### Общий WebGL контекст

Браузеры ограничивают число одновременных WebGL контекстов (~16), поэтому при 100+
виджетах нельзя создавать контекст на экземпляр. Используется один общий
`SharedRenderEngine`: сцена каждого виджета рендерится в общую WebGL канву
и копируется (blit) на собственную 2D канву виджета.

### Элементы отрисовки

Отрисовка разделена на изолированные элементы (`render/elements/*`), каждый отвечает
за одну часть сцены и обновляется только при изменении своих данных
(`interestMask` из `DirtyFlags`):

| Элемент | Отвечает за |
| --- | --- |
| `RowBackgroundsElement` | фоны строк (ask/bid/mixed/spread, лучшие цены, hover) |
| `TableGridElement` | minor/major линии сетки и разделители колонок |
| `VolumeColumnElement` | объемы, подсветка объемов, накопленный объем |
| `PriceColumnElement` | цены, диапазон позиции, рамка строк с заявками |
| `OrdersColumnElement` | индикаторы собственных заявок, drag-призрак, hit-области |
| `TradesPanelElement` | панель сделок (эллипсы, связи, ghost/out-of-range, свои сделки) |
| `ClustersPanelElement` | кластеры сделок (агрегация по строкам, подсветка, скролл) |

### События вместо торговой логики

Рендер не выставляет заявки. Он сообщает о действиях пользователя через колбэки
(`ScalperOrderBook2RendererEvents`): нажатие на строку (цена + тип строки),
клик/перетаскивание индикатора заявок, hover, изменение видимого диапазона,
приближение к краю ценового ряда. Команды выполняет Angular-слой
(`scalper-order-book-2-surface`) через переиспользуемый `ScalperCommandProcessorService`.

### HTML оверлеи

Редко обновляющиеся элементы накладываются поверх канвы обычным HTML
(переиспользуются компоненты `scalper-order-book`): верхняя панель, плавающие панели,
панель текущей позиции, рабочие объемы, short/long индикаторы, индикаторы заявок вне
экрана, панель возможных действий, линейка (маркер), ручки изменения ширины секций,
контекстное меню кластеров.

## Структура

| Папка | Назначение |
| --- | --- |
| `render/` | Изолированный слой отрисовки (pixi.js, без Angular и rxjs) |
| `render/elements/` | Элементы отрисовки |
| `components/scalper-order-book-2/` | Корневой компонент: shared context + команды |
| `components/scalper-order-book-2-body/` | Data context, компоновка, HTML оверлеи |
| `components/scalper-order-book-2-surface/` | Мост: data context -> фасад рендера, события -> команды |
| `utils/` | Хелперы потока данных (кластеры, собственные сделки) |

## Настройки

Тип настроек — alias `ScalperOrderBookWidgetSettings`. Используются те же
двухуровневые настройки (instance + shared per-instrument), та же форма настроек
и то же локальное состояние (`WidgetLocalStateService`): масштаб, рабочий объем,
позиции плавающих панелей.
