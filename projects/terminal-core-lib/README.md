Данный проект содержит общие компоненты, пайты, директивы, типы, сервисы,  утилиты и  assets которые доступны всем приложениям.

Папка assets содержит общее содержимое, которые будет доступно по пути /assets из приложения (см. angular.json), используемое объявленными в проекте сервисами. 

Папка common содержит общие компоненты, пайты, директивы, типы, сервисы, утилиты, которые используются повсеместно в приложениях и которые не формируют отдельную фичу (feature).

Папка config содержит определения типов и injection токенов, которые соответствуют каким-либо конфигурациям.

Папка features включается в себя функциональные блоки.
Такие блоки могут включать типы, сервисы, компоненты, директивы, пайпы, утилиты, определения сторов и т.д.
Т.е. feature представляет собой набор файлов, cвязанных общей функциональностью.

Каждая папка в features соответсвует отдельной feature.
В свою очередь файлы в папке feature должны быть распределены по подпапкам: components, pipes, directives, services, types, utils, store, hooks (файлы, реализующие интерфейс Hook).

Предпочтительно чтобы сервисы имели аттрибут @Injectable({providedIn: 'root''}).
Однако если это сделать невозможно, например, сервит инжектит токен, который не инжектится в root, то необходимо провайдеры объявить в отдельном файле с суффиксом ".providers" (см. projects/terminal-core-lib/src/features/terminal-settings/terminal-settings-storage.providers.ts для примера).

ВАЖНО: Данный проект не должен включать импортов других проектов в репозитории.
