0. Использовать best practice Angular 21
1. Нужно использовать signals где это возможно, где ui реагирует ни изменение какой-либо переменной типа isLoading. Избегать подписок на observable чтобы положить значение в синхронную переменную класса. Это-за это change-detection отрабатывает плохо
2. Использолвать flex вместо grid - не стоит миксовать подходы в одном проекте
3. По максимуму использовать хелперы стилей из \styles\utils. прописывание одних и тех же стилей для каждого компонента увеличивает начальный размер загрузки, а соответственно. время старта приложения
4. Не использовать устаревшие ngif, ngFor, ngSwitch. Они уже удалялись раз и опять воскресли в этой ветке. При необходимости есть миграция https://angular.dev/reference/migrations/control-flow
5. Не нужно создавать заготовки тестов. Их автоматическое создание уже было выключено в angular.json, не понимаю откуда они опять воскресли. Эти тесты не делают ничего полезного, а только забирают время на поддержку. При каждом добавлении зависимости нужно ее еще и в тесты прописывать и создавать моки. При рефакторинге эти моки нужно обновлять
6. Не добавлять в компоненты зависимости с названиеи переменной service. Каждый сервис должен иметь четкое имя, а не общее
7. Использовать типизированный формы и не читать контролы по строковым именам
8. Не прописывать у компонентов standalone: true - явный признак LLM. Это было нужно в прошлой версии angular
9. По максимуму изолировать особенности сервиса платежа в самом платеже. Astras должен работать со своими контрактами, а не знать осовенности конкретной реализации
10. Если сервис и контракты используются только внутри модуля, то добавлять их в shared нет смысла
11. Все переводы должны включать перевод на армянский чтобы потом не искать какие переводы отсутствуют. На армянском контуре итак полный зоопарк из-за рассинхрона файлов
